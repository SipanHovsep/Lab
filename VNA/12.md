## Abstract
In high-frequency electronics, the Vector Network Analyzer (VNA) is the primary instrument for characterizing the stochastic and deterministic behavior of RF components. However, raw data acquired at the VNA’s test ports is rarely a true representation of the Device Under Test (DUT) due to systematic errors. This article details a Python-based implementation of a 12-term error model designed to shift the measurement reference plane through a 10dB attenuator and associated cabling. By de-embedding these components, we reveal the fundamental S-parameters of the DUT with high precision.

---

## I. Introduction: Target Audience and Usefulness

By default, a VNA measures at its front-panel connectors. To obtain accurate data, engineers must mathematically "shift" the reference plane to the DUT interface. This process requires identifying and removing systematic measurement errors—such as directivity, source match, and reflection tracking—that would otherwise mask the true performance of the component.

This article is primarily intended for undergraduate and graduate engineering students, as well as junior RF engineers seeking a deeper mathematical intuition for vector network analysis. By providing a transparent, step-by-step implementation, this work serves as a practical bridge between Signal Flow Graph analysis and the software-driven measurements encountered in modern laboratories, allowing readers to apply these concepts to their own custom de-embedding challenges.

## II. Motivation: Beyond the "Black Box"
In modern commercial VNAs, error correction is often a "one-touch" process facilitated by automated calibration modules (ECal) and proprietary firmware. While this efficiency is essential for laboratory throughput, it often renders the underlying mathematical transformations opaque to the user.

The primary objective of this project is pedagogical. Relying on a VNA as a "black box" is sufficient for daily productivity, but mastering the mathematics behind the measurement is essential for advanced RF engineering. By manually coding error-correction algorithms, we can "look under the hood" of the calibration engine and observe how raw, uncorrected measurements are transformed into clean, usable data.

While existing literature typically covers abstract algebraic proofs or 'how-to' guides for vendor-specific VNA hardware, this work offers a middle ground through an open-source, reproducible computational framework. By utilizing the scikit-rf library to process real-world Touchstone data, it provides a functional template that moves beyond textbook idealizations into the realm of practical, scriptable computation.

This work is inspired by the contributions of Brian Walker of Copper Mountain Technologies, who provided a clear and practical treatment of one-port VNA calibration using the three-term error model @walker2014one. While Walker’s work focuses on single-port reflection measurements, the present study extends this framework to the two-port, twelve-term error model. This extension is implemented in a Python-based computational environment to manage the increased algebraic and matrix complexity involved in full two-port error correction.

## III. Analytical Framework and Toolchain
To replicate the VNA’s internal calibration engine, this implementation synthesizes classical RF literature with modern data science tools:

* **Mathematical Foundation:** The formulation follows Douglas Rytting’s 2003 ARFTG paper, “Network Analyzer Error Models and Calibration Methods” @rytting2003network, which defines the full 12-term error model and its associated calibration procedures.
* **Computational Environment:** Python is used to implement the calibration routines, providing native support for complex-valued numerical operations and matrix-based computation.
* **Data Handling:** The **scikit-rf (skrf)** @arsenovic2022skrf library was utilized for data ingestion. Rather than developing a custom parser for Touchstone files, `scikit-rf` was used to ingest raw `.s1p` and `.s2p` files, allowing the project to focus entirely on the implementation of the error-correction math.

## Methodology

The logic for acquiring the directivity, source match, and reflection tracking terms is identical to the standard 3-term error model. Because the 12-term model relies on independent forward and reverse models, these initial terms are extracted by performing a standard 1-port calibration (using Open, Short, and Load standards) on Port 1 and Port 2 separately. The remaining transmission terms (load match, transmission tracking, and isolation) are then solved using the Thru measurement.


* **Step 1: One-Port Calibration on Port 1**
  * Calibrate Port-1 using the standard one-port procedure.
  * Solve for $e_{11}$, $e_{00}$, and $(e_{10}e_{01})$.
  * Calculate $(e_{10}e_{01})$ from $\Delta_e$.

* **Step 2: Isolation Measurement**
  * Connect $Z_0$ terminations to both Ports 1 and 2.
  * Measure $S_{21M}$, which provides the isolation term ($e_{30}$) directly.

* **Step 3: Thru Measurement**
  * Connect Ports 1 and 2 together.
  * Calculate the load match ($e_{22}$) using:
    $$e_{22} = \frac{S_{11M} - e_{00}}{S_{11M}e_{11} - \Delta_e}$$
  * Calculate the transmission tracking ($e_{10}e_{32}$) using:
    $$e_{10}e_{32} = (S_{21M} - e_{30})(1 - e_{11}e_{22})$$

* **Step 4: Reverse Model**
  * Use the exact same process for the reverse model to extract the remaining 6 terms.

Using `scikit-rf`, the raw Touchstone files acquired from the setups above are imported and converted into Network objects, and calculations shown in {numref}`fig-forward` are performed.
```python
import skrf as rf
import matplotlib.pyplot as plt
import numpy as np
import math

numpoints=1601;
#ideals
openGamma=rf.Network('Port1/ideals/open.s1p')
shortGamma=rf.Network('Port1/ideals/short.s1p')
loadGamma=rf.Network('Port1/ideals/load.s1p')
#measured
m_openGamma=rf.Network('Port1/measured/open.s1p')
m_shortGamma=rf.Network('Port1/measured/short.s1p')
m_loadGamma=rf.Network('Port1/measured/load.s1p')

# Gamma Actual

open_Gamma=openGamma.s
short_Gamma=shortGamma.s
load_Gamma=loadGamma.s

Gamma_a1=np.zeros((numpoints),dtype=complex)
Gamma_a2=np.zeros((numpoints),dtype=complex)
Gamma_a3=np.zeros((numpoints),dtype=complex)


Gamma_m1=np.zeros((numpoints),dtype=complex)
Gamma_m2=np.zeros((numpoints),dtype=complex)
Gamma_m3=np.zeros((numpoints),dtype=complex)


for i in range(0,numpoints):
    Gamma_a1[i]=complex(open_Gamma[i,0,0].real,open_Gamma[i,0,0].imag)
    Gamma_a2[i]=complex(short_Gamma[i,0,0].real,short_Gamma[i,0,0].imag)
    Gamma_a3[i]=complex(load_Gamma[i,0,0].real,load_Gamma[i,0,0].imag)
# Gamma Measured

m_open_Gamma=m_openGamma.s
m_short_Gamma=m_shortGamma.s
m_load_Gamma=m_loadGamma.s

for i in range(0,numpoints):
    Gamma_m1[i]=complex(m_open_Gamma[i,0,0].real,m_open_Gamma[i,0,0].imag)
    Gamma_m2[i]=complex(m_short_Gamma[i,0,0].real,m_short_Gamma[i,0,0].imag)
    Gamma_m3[i]=complex(m_load_Gamma[i,0,0].real,m_load_Gamma[i,0,0].imag)

A=np.zeros((3,3),dtype=complex)
B=np.zeros((3,3),dtype=complex)
X=np.zeros((3,1),dtype=complex)

e_00=np.zeros((numpoints),dtype=complex) # Directivity
e_11=np.zeros((numpoints),dtype=complex) #Source match
delta_e=np.zeros((numpoints),dtype=complex) #Delta e
e_10_e_01=np.zeros((numpoints),dtype=complex) #Reflection Tracking 

for i in range (0,numpoints):
    A[0,0]=1 
    A[0,1]=Gamma_a1[i]*Gamma_m1[i]
    A[0,2]=-Gamma_a1[i]
    A[1,0]=1
    A[1,1]=Gamma_a2[i]*Gamma_m2[i]
    A[1,2]=-Gamma_a2[i]
    A[2,0]=1
    A[2,1]=Gamma_a3[i]*Gamma_m3[i]
    A[2,2]=-Gamma_a3[i]
    
    B[0]=Gamma_m1[i]
    B[1]=Gamma_m2[i]
    B[2]=Gamma_m3[i]
    
    
    A_inv=np.linalg.inv(A)
    X=np.dot(A_inv,B)
    
    e_00[i]=X[0,0]
    e_11[i]=X[1,0]
    delta_e[i]=X[2,0]
    e_10_e_01[i]=e_00[i]*e_11[i]-delta_e[i]
    

np.save("variables/e_00", e_00)
np.save("variables/e_11", e_11)
np.save("variables/e_10_e_01", e_10_e_01)
```

The same process was repeated for Port 2. 

After this step, we acquire 6 out of the 12 error terms:

* $e_{00}$ (Forward Directivity)
* $e_{11}$ (Forward Source Match)
* $e_{10}e_{01}$ (Forward Reflection Tracking)
* $\bar{e}_{33}$ (Reverse Directivity)
* $\bar{e}_{11}$ (Reverse Source Match)
* $\bar{e}_{23}\bar{e}_{32}$ (Reverse Reflection Tracking)

```python
import skrf as rf
import matplotlib.pyplot as plt
import numpy as np
import math

numpoints=1601;

e_00=np.load("variables/e_00.npy")
e_11=np.load("variables/e_11.npy")
e_10_e_01=np.load("variables/e_10_e_01.npy")
e_30=np.load("variables/e_30.npy")
delta_e=np.zeros((numpoints),dtype=complex)


for i in range(0,numpoints):
    delta_e[i]=(e_00[i]*e_11[i])-e_10_e_01[i]


s11M=rf.Network('step3/s11M.s1p')
s12M=rf.Network('step3/s12M.s1p')
s21M=rf.Network('step3/s21M.s1p')
s22M=rf.Network('step3/s22M.s1p')

s11M=s11M.s
s12M=s12M.s
s21M=s21M.s
s22M=s22M.s

s11_M=np.zeros((numpoints),dtype=complex)
s12_M=np.zeros((numpoints),dtype=complex)
s21_M=np.zeros((numpoints),dtype=complex)
s22_M=np.zeros((numpoints),dtype=complex)


for i in range(0,numpoints):
    s11_M[i]=complex(s11M[i,0,0].real,s11M[i,0,0].imag)
    s12_M[i]=complex(s12M[i,0,0].real,s12M[i,0,0].imag)
    s21_M[i]=complex(s21M[i,0,0].real,s21M[i,0,0].imag)
    s22_M[i]=complex(s22M[i,0,0].real,s22M[i,0,0].imag)

e_22=np.zeros((numpoints),dtype=complex)
e_10_e_32=np.zeros((numpoints),dtype=complex)

for i in range (0,numpoints):
    e_22[i]=(s11_M[i]-e_00[i])/(s11_M[i]*e_11[i]-delta_e[i])
    e_10_e_32[i]=(s21_M[i]-e_30[i])*(1-e_11[i]*e_22[i])


np.save("variables/e_22", e_22)
np.save("variables/e_10_e_32", e_10_e_32)
```

The similar procedure is repeated for Reverse model. 

After this step, we got all 4 remaining error terms:

* $e_{22}$ (Forward Load Match)
* $e_{10}e_{32}$ (Forward Transmission Tracking)
* $\bar{e}_{22}$ (Reverse Load Match)
* $\bar{e}_{23}\bar{e}_{01}$ (Reverse Transmission Tracking)

## Calculation

After acquiring, we load 12 error terms to do final set of calculations for correction, shown in {numref}`fig-formulas`

```python
import skrf as rf
import matplotlib.pyplot as plt
import numpy as np
import math

numpoints=1601;

#load s Measured

s11M=rf.Network('step3/s11M.s1p')
s12M=rf.Network('step3/s12M.s1p')
s21M=rf.Network('step3/s21M.s1p')
s22M=rf.Network('step3/s22M.s1p')

s11M=s11M.s
s12M=s12M.s
s21M=s21M.s
s22M=s22M.s

s11_M=np.zeros((numpoints),dtype=complex)
s12_M=np.zeros((numpoints),dtype=complex)
s21_M=np.zeros((numpoints),dtype=complex)
s22_M=np.zeros((numpoints),dtype=complex)


s11=np.zeros((numpoints),dtype=complex)
s12=np.zeros((numpoints),dtype=complex)
s21=np.zeros((numpoints),dtype=complex)
s22=np.zeros((numpoints),dtype=complex)


for i in range(0,numpoints):
    s11_M[i]=complex(s11M[i,0,0].real,s11M[i,0,0].imag)
    s12_M[i]=complex(s12M[i,0,0].real,s12M[i,0,0].imag)
    s21_M[i]=complex(s21M[i,0,0].real,s21M[i,0,0].imag)
    s22_M[i]=complex(s22M[i,0,0].real,s22M[i,0,0].imag)



# load 12 terms
e_22_bar=np.load("variables/e_22_bar.npy")
e_23_bar_e_01_bar=np.load("variables/e_23_bar_e_01_bar.npy")
e_10_e_32=np.load("variables/e_10_e_32.npy")
e_22=np.load("variables/e_22.npy")
e_00=np.load("variables/e_00.npy")
e_10_e_01=np.load("variables/e_10_e_01.npy")
e_11=np.load("variables/e_11.npy")
e_03_bar=np.load("variables/e_03_bar.npy")
e_30=e_03_bar=np.load("variables/e_30.npy")
e_11_bar=np.load("variables/e_11_bar.npy")
e_23_bar_e_32_bar=np.load("variables/e_23_bar_e_32_bar.npy")
e_33_bar=np.load("variables/e_33_bar.npy")

D=np.zeros((numpoints),dtype=complex)



for i in range(0,numpoints):
    D[i] = (
    (1 + ((s11_M[i] - e_00[i]) / e_10_e_01[i]) * e_11[i]) *
    (1 + ((s22_M[i] - e_33_bar[i]) / e_23_bar_e_32_bar[i]) * e_22_bar[i]) -
    ((s21_M[i] - e_30[i]) / e_10_e_32[i]) *
    ((s12_M[i] - e_03_bar[i]) / e_23_bar_e_01_bar[i]) *
    e_22[i] * e_11_bar[i]
)         


for i in range(0,numpoints):
    s21[i] = (
    (((s21_M[i] - e_30[i])/ e_10_e_32[i])*(1+((s22_M[i] - e_33_bar[i]) / e_23_bar_e_32_bar[i])*(e_22_bar[i]-e_22[i])))/D[i]
)         




for i in range(0,numpoints):
    s21[i]=20*math.log10(abs(s21[i]))
    s21_M[i]=20*math.log10(abs(s21_M[i]))
    
plt.figure()


plt.plot(s21, label='Corrected S21')
plt.plot(s21_M, label='Raw S21')
plt.legend()
```
